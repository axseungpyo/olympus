# RP-030: Cowork Dashboard — 최종 통합 UI

## Summary
Overview 탭을 cowork 중심 레이아웃으로 재구성하고, `useCoworkState` 훅으로 상태/진행률/Odin 메시지 WebSocket 데이터를 통합했다.

## Context Digest (for Odin)
- Architecture: Overview 전용 cowork UI 상태를 `useCoworkState`로 분리했고, 기존 나머지 대시보드 탭 데이터 흐름은 유지했다.
- Key entry points: `dashboard/app/page.tsx`, `dashboard/hooks/useCoworkState.ts`, `server/websocket/index.ts`
- Config/env: WebSocket URL은 기존 `getWsBase()`와 bearer token 쿼리 파라미터를 그대로 사용한다.
- Gotchas: `api/health` 검증은 현재 샌드박스에서 포트 listen이 금지되어 PASS로 만들 수 없었다.
