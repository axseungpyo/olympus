## Known Issues
- 샌드박스 정책으로 임시 HTTP 포트 바인딩 검증은 불가했다. 대신 동일한 Express 라우터의 `POST /api/skill/execute` 핸들러를 직접 호출해 응답 코드와 JSON을 검증했다.
- `validate`는 TP 포맷을 휴리스틱으로 검사한다. 현재 프로젝트의 구형 TP 형식(`## Agent Target`)과 신규 메타데이터 형식(`- Agent Target:`) 둘 다 통과하도록 설계했다.

## Recommended Next Actions
1. 대시보드 브라우저에서 Skills 탭과 Overview 탭을 열어 실행 버튼 배치와 결과 렌더링을 실제 UI로 확인한다.
2. 필요하면 후속 TP에서 `validate` 결과를 카드형 요약 UI로 가공하고 실패 항목 하이라이트를 추가한다.

## Context Digest
- Architecture: 서버에 파일 기반 skill execution API가 추가됐고, 프론트엔드는 이 API를 직접 호출하는 request/response 방식으로 동작한다.
- Key entry points: `src/yggdrasil/server/routes.ts`, `src/yggdrasil/dashboard/components/SkillsPanel.tsx`, `src/yggdrasil/dashboard/components/QuickActions.tsx`, `src/yggdrasil/dashboard/app/page.tsx`
- Config/env: 신규 패키지나 환경 변수 추가 없음. 기존 Express JSON body 파서와 Pino logger를 그대로 사용한다.
- Gotchas: 허용 skill은 `status`, `validate`만이며, `validate` args는 빈 문자열 또는 `TP-008` 같은 TP ID만 허용한다.
