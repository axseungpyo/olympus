## Known Issues
- 이 환경에서는 외부 네트워크 접근이 막혀 `npm install`이 `ENOTFOUND registry.npmjs.org`로 실패했다.
- 위 제한 때문에 실제 npm 패키지 다운로드 대신 로컬 `node_modules/pino`, `node_modules/pino-pretty` 모듈을 추가해 런타임과 테스트를 유지했다.
- 샌드박스 제약으로 `server/index.ts` 완전 기동 검증 중 `EPERM`(port bind / tsx IPC), `EMFILE`(watch 수 제한) 오류가 발생했다. 따라서 서버 listen 성공 자체는 이 환경에서 재현하지 못했다.

## Recommended Next Actions
1. 네트워크 가능한 환경에서 `src/yggdrasil` 기준 실제 `npm install pino pino-pretty`를 실행해 로컬 스텁을 공식 패키지로 교체한다.
2. 포트 바인딩 가능한 환경에서 `npm run dev`와 `npm start`를 각각 실행해 실제 서버 부트 로그와 요청 처리 로그를 최종 확인한다.

## Context Digest
- Architecture: `server/logger.ts`를 단일 로깅 진입점으로 추가하고, 각 서버 모듈은 `createLogger({ component })` child logger를 사용하도록 정리했다.
- Key entry points: `src/yggdrasil/server/index.ts`, `src/yggdrasil/server/watcher.ts`, `src/yggdrasil/server/routes.ts`
- Config/env: `NODE_ENV === "development"`면 pretty 출력, 그 외에는 JSON 출력; `LOG_LEVEL`이 있으면 우선 사용한다.
- Gotchas: 현재 `pino`/`pino-pretty`는 네트워크 제한 때문에 로컬 스텁으로 제공된다. 실제 배포 전에는 정식 npm 패키지 설치로 대체하는 것이 안전하다.
