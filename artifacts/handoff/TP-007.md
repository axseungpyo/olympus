# TP-007: Yggdrasil 실행 흐름 시각화 + 통계 대시보드

## Metadata
- Agent Target: codex
- Complexity: complex
- Priority: P2
- Created: 2026-03-09

## Objective

Yggdrasil 대시보드에 TP→Agent→RP 파이프라인 시각화 컴포넌트와 태스크 통계 대시보드를 추가한다.

## Scope In

### 1. FlowView 컴포넌트 (실행 흐름 시각화)
- `dashboard/components/FlowView.tsx` 생성
- 각 태스크를 TP(Rune) → Agent(실행) → RP(Saga) 3단계로 시각화
- 상태별 색상 표시 (constants.ts의 TASK_STATUS_CONFIG 사용)
- CSS/SVG 기반 노드+연결선 (외부 차트 라이브러리 설치 금지)
- 현재 in-progress 태스크는 애니메이션 표시

### 2. StatsPanel 컴포넌트 (통계 대시보드)
- `dashboard/components/StatsPanel.tsx` 생성
- 표시 항목: 총 태스크 수, 완료율, 에이전트별 태스크 수, 상태별 분포
- CSS 기반 바 차트 (외부 차트 라이브러리 설치 금지)
- tasks prop에서 실시간 계산

### 3. page.tsx 통합
- ViewMode에 `"flow"`, `"stats"` 탭 추가
- 기존 overview/terminals/skills 탭 유지

## Scope Out
- 외부 라이브러리(D3, Chart.js, Recharts 등) 설치 금지 — CSS/SVG만 사용
- 서버(server/) 코드는 변경하지 않는다
- 기존 테스트는 수정하지 않는다
- 태스크 소요시간 계산은 하지 않는다 (created/updated만 사용)

## Relevant Files
- `src/yggdrasil/dashboard/app/page.tsx`
- `src/yggdrasil/dashboard/lib/constants.ts`
- `src/yggdrasil/dashboard/lib/types.ts`
- `src/yggdrasil/dashboard/components/` (기존 컴포넌트 참고)

## Acceptance Criteria
1. `FlowView.tsx` 파일이 존재하고 tasks prop을 받아 파이프라인 시각화를 렌더링한다
2. `StatsPanel.tsx` 파일이 존재하고 tasks prop을 받아 통계를 표시한다
3. page.tsx의 ViewMode에 `"flow"`와 `"stats"` 탭이 추가되어 있다
4. 상태별 색상이 `TASK_STATUS_CONFIG`에서 가져온다
5. 새 npm 패키지가 설치되지 않았다 (package.json에 변경 없음)
6. `npm test` 20개 테스트 전부 PASS한다
7. TypeScript 컴파일 에러가 없다

## Implementation Notes
- FlowView: 각 태스크를 가로 3칸(TP→Agent→RP)으로 배치. 연결선은 CSS border 또는 SVG line으로 구현.
- StatsPanel: `useMemo`로 통계 계산. flex 기반 가로 바 차트.
- 색상은 반드시 `TASK_STATUS_CONFIG`, `AGENT_CONFIG`에서 import.
- page.tsx의 탭 배열: `["overview", "terminals", "flow", "stats", "skills"]`
