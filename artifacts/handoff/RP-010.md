# RP-010: 롤백 메커니즘 — 실패 시 git revert 기반 자동 롤백

## Summary

`scripts/rollback.sh`를 추가해 TP 단위 커밋 검색, dry-run, 역순 `git revert --no-commit`, 단일 revert 커밋 생성, `artifacts/logs/rollback.log` 기록 흐름을 구현했다.

## Known Issues
- `scripts/rollback.sh`는 현재 작업 트리가 깨끗하다는 전제에서 가장 안전하다. 미커밋 변경이 있으면 `git revert` 자체가 실패할 수 있으며, 이 경우 git 기본 오류에 따라 수동 정리가 필요하다.
- rejected RP 감지는 `RP_FILE` 내 `rejected` 문자열 검색 기반이다. 리뷰 포맷이 바뀌면 탐지 방식도 같이 조정해야 한다.

## Recommended Next Actions

2. Odin 운영 문서나 runbook에 `/rollback TP-NNN` 사용 시점과 수동 판단 기준을 명시한다.

## Context Digest
- Architecture: rollback은 별도 오케스트레이터 없이 git log/revert를 직접 호출하는 독립 shell 유틸리티로 추가됐다.
- Key entry points: `scripts/rollback.sh`, `scripts/delegate-codex.sh`, `.claude/skills/rollback/SKILL.md`
- Config/env: 추가 패키지 설치는 없고, macOS 기본 Bash 3.x 호환을 위해 `mapfile` 없이 구현했다.
- Gotchas: 실제 rollback은 자동 실행되지 않으며, `--dry-run`으로 대상 커밋을 먼저 확인한 뒤 Odin이 명시적으로 실행해야 한다.
