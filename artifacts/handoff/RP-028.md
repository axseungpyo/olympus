# RP-028: 병렬 에이전트 실행 + 실시간 진행 스트리밍

## Summary
Planner가 `parallel` step 그룹을 `Promise.all`로 실행하도록 변경했고, 에이전트 프로세스 stdout을 모니터링해 진행률을 `agent.progress` 이벤트로 발행하도록 연결했다.
대시보드는 `agent_progress` 웹소켓 메시지를 받아 overview에서 에이전트별 진행 바를 실시간 표시하며, 완료/실패 항목은 5초 후 제거된다.

## Context Digest (for Odin)
- Architecture: Planner는 step 그룹 단위로 순차 진행하되, 그룹 내부 `parallel=true` step은 병렬 실행한다.
- Key entry points: `StartAgentUseCase`, `MonitorAgentUseCase`, `PlannerUseCase`, `websocket/index.ts`, dashboard overview.
- Config/env: 추가 환경변수는 없고 기존 `AGENT_MODE` 흐름을 그대로 사용한다.
- Gotchas: 진행률은 stdout 기반 추정치라 실제 완료율이 아니라 활동량 기반이며, 최대 95% 후 종료 이벤트에서 100%로 마감된다.
