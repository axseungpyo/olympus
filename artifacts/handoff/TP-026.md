## Objective

Claude(Odin)가 사용자의 자연어 요청을 **멀티스텝 실행 계획으로 분해**하고,
각 단계를 순차적으로 실행하며, 실패 시 재시도하는 **Planner 엔진**을 구축한다.

예: "소셜 로그인 추가해줘" → 분석 → TP 생성 → Brokkr 위임 → RP 검토 → 완료

## Context

### 현재 상태

```
사용자 → ProcessCommandUseCase → LLM 응답 (tool_use 1-3개) → 즉시 실행
```

- 단일 턴에서 최대 3개 Tool만 호출 가능
- **멀티스텝 워크플로우 없음** — 계획 수립/순차 실행/의존관계/재시도 없음
- Claude가 "분석 → TP → 위임 → 검토"를 자율적으로 이어갈 수 없음

### 목표 구조

```
사용자 → ProcessCommandUseCase
  ↓ (Claude가 create_plan tool 호출)
  PlannerUseCase.createPlan()
  ↓
  ExecutionPlan { steps: PlanStep[] }
  ↓
  PlannerUseCase.executePlan()
    Step 1: search_codebase → 코드 분석
    Step 2: create_task → TP 생성
    Step 3: delegate_task → Brokkr 위임 (승인)
    Step 4: review_saga → RP 검토
  ↓
  진행률 → EventBus → WebSocket → 대시보드
```

  action: string;         // tool name: "search_codebase" | "create_task" | "delegate_task" | "review_saga" | "read_file" | "write_file"

  error?: string;
}

  status: PlanStatus;

}

### 5. core/use-cases/plan/PlannerUseCase.ts (신규, ~150줄)

핵심 로직:

```typescript
export class PlannerUseCase {
  constructor(
    private readonly planRepository: IPlanRepository,
    private readonly toolExecutors: IToolExecutor[],
    private readonly approvalStore: IApprovalStore,
    private readonly messageRepository: IMessageRepository,
    private readonly eventBus: IEventBus,
    private readonly projectRoot: string,
  ) {}

  async createPlan(goal: string, steps: PlanStep[]): Promise<ExecutionPlan> {
    const plan: ExecutionPlan = {
      id: `plan-${Date.now()}`,
      goal,
      steps: steps.map((s, i) => ({ ...s, order: i + 1, status: "pending" })),
      status: "draft",
      currentStep: 0,
      maxRetries: 3,
      retryCount: 0,
      createdAt: Date.now(),
      updatedAt: Date.now(),
    };
    await this.planRepository.save(plan);
    return plan;
  }

  async executePlan(planId: string): Promise<ExecutionPlan> {
    const plan = await this.planRepository.getById(planId);
    if (!plan) throw new Error(`Plan not found: ${planId}`);

    plan.status = "running";
    await this.planRepository.update(plan);

    for (const step of plan.steps) {
      if (step.status === "completed" || step.status === "skipped") continue;

      // 의존성 체크
      if (step.dependsOn?.length) {
        const unmet = step.dependsOn.filter(dep => {
          const depStep = plan.steps.find(s => s.order === dep);
          return depStep && depStep.status !== "completed";
        });
        if (unmet.length > 0) {
          step.status = "skipped";
          step.error = `의존 단계 미완료: ${unmet.join(", ")}`;
          continue;
        }
      }

      // 승인 필요 단계 → 중단하고 대시보드에 표시
      if (step.requiresApproval) {
        plan.currentStep = step.order;
        plan.status = "paused";
        this.publishProgress(plan, step);
        await this.planRepository.update(plan);
        return plan; // 승인 대기
      }

      // 실행
      step.status = "running";
      plan.currentStep = step.order;
      this.publishProgress(plan, step);

      const executor = this.toolExecutors.find(e => e.canHandle(step.action));
      if (!executor) {
        step.status = "failed";
        step.error = `실행기 없음: ${step.action}`;
        continue;
      }

      const result = await executor.execute(
        { name: step.action, input: step.input },
        this.projectRoot,
      );

      if (result.success) {
        step.status = "completed";
        step.result = result.output;
      } else {
        step.status = "failed";
        step.error = result.error;
        plan.retryCount++;
        if (plan.retryCount >= plan.maxRetries) {
          plan.status = "failed";
          await this.planRepository.update(plan);
          return plan;
        }
        // 재시도: step 상태 리셋
        step.status = "pending";
      }

      this.publishProgress(plan, step);
      await this.planRepository.update(plan);
    }

    plan.status = plan.steps.every(s => s.status === "completed" || s.status === "skipped")
      ? "completed"
      : "failed";
    plan.updatedAt = Date.now();
    await this.planRepository.update(plan);
    return plan;
  }

  async getActivePlan(): Promise<ExecutionPlan | null> {
    return this.planRepository.getActive();
  }

  async resumePlan(planId: string): Promise<ExecutionPlan> {
    // 승인 후 재실행
    return this.executePlan(planId);
  }

  private publishProgress(plan: ExecutionPlan, step: PlanStep): void {
    this.eventBus.publish({
      type: PLAN_PROGRESS_EVENT,
      timestamp: Date.now(),
      payload: {
        planId: plan.id,
        goal: plan.goal,
        currentStep: step.order,
        totalSteps: plan.steps.length,
        stepDescription: step.description,
        stepStatus: step.status,
      },
    });
  }
}
```

### 7. adapters/tools/PlannerToolExecutor.ts (신규, ~60줄)

IToolExecutor 구현. create_plan/get_plan_status 처리:

```typescript
export class PlannerToolExecutor implements IToolExecutor {
  constructor(private readonly plannerUseCase: PlannerUseCase) {}

  canHandle(toolName: string): boolean {
    return ["create_plan", "get_plan_status"].includes(toolName);
  }

  async execute(toolCall: LLMToolCall, projectRoot: string): Promise<ToolResult> {
    if (toolCall.name === "create_plan") {
      const plan = await this.plannerUseCase.createPlan(
        toolCall.input.goal as string,
        toolCall.input.steps as PlanStep[],
      );
      const executed = await this.plannerUseCase.executePlan(plan.id);
      return this.formatPlanResult(executed);
    }

    if (toolCall.name === "get_plan_status") {
      const active = await this.plannerUseCase.getActivePlan();
      if (!active) return { success: true, output: "활성 실행 계획이 없습니다." };
      return this.formatPlanResult(active);
    }

    return { success: false, output: "", error: "Unknown tool" };
  }

  private formatPlanResult(plan: ExecutionPlan): ToolResult {
    const lines = [
      `Plan: ${plan.goal} [${plan.status}]`,
      `Progress: ${plan.steps.filter(s => s.status === "completed").length}/${plan.steps.length}`,
      ...plan.steps.map(s => `  ${s.order}. [${s.status}] ${s.description}${s.error ? ` — ${s.error}` : ""}`),
    ];
    return { success: true, output: lines.join("\n") };
  }
}
```

## Acceptance Criteria

1. **AC-1**: `create_plan` tool — goal + steps로 ExecutionPlan 생성
2. **AC-2**: Plan 실행 — steps를 순차적으로 실행하며 각 step의 status 업데이트
3. **AC-3**: 의존관계 — dependsOn이 완료되지 않은 step은 skip
4. **AC-4**: 재시도 — step 실패 시 재시도, maxRetries(3) 초과 시 plan failed
5. **AC-5**: 승인 대기 — requiresApproval step에서 plan paused 반환
6. **AC-6**: `get_plan_status` tool — 활성 계획의 진행 상태 반환
7. **AC-7**: `PLAN_PROGRESS` 이벤트가 EventBus에 발행됨
8. **AC-8**: WebSocket으로 plan_progress 이벤트 전달
