## Objective
모놀리식 `routes.ts` (711줄, 25개 핸들러)를 6개 도메인별 라우터 파일로 분리하고, `routes/index.ts`에서 조합한다. 기능 변경 없이 구조만 변경.

## Scope In
- `src/yggdrasil/server/routes/` 디렉토리 생성
- 기존 `routes.ts`의 25개 핸들러를 6개 라우터 파일로 분리:
  1. `routes/health.routes.ts` — GET /api/health, GET /api/status, GET /api/metrics (3개)
  2. `routes/agent.routes.ts` — GET /api/agent/:name/health, POST /api/agent/:name/start, POST /api/agent/:name/stop, GET /api/agent/modes (4개)
  3. `routes/task.routes.ts` — GET /api/tasks, POST /api/tasks, GET /api/tasks/:id, PUT /api/tasks/:id/status, DELETE /api/tasks/:id (5개)
  4. `routes/odin.routes.ts` — GET /api/odin/messages, POST /api/odin/command, POST /api/odin/approve (3개)
  5. `routes/mcp.routes.ts` — GET /api/mcp/servers, POST /api/mcp/servers, PUT /api/mcp/servers/:name, DELETE /api/mcp/servers/:name, POST /api/mcp/sync (5개)
  6. `routes/document.routes.ts` — GET /api/chronicle, GET /api/docs/:id, POST /api/skills/:name/execute, GET /api/dependency-graph, GET /api/agents (5개)
- `routes/index.ts` — 6개 라우터를 조합하고 authMiddleware를 적용
- 기존 `server/routes.ts` 삭제
- `server/index.ts`의 import를 `./routes` → `./routes/index`로 변경

## Implementation Notes
1. 각 라우터 파일은 `export function createXxxRouter(asgardRoot: string): Router` 패턴 사용.
2. `authMiddleware`는 `routes/index.ts`에서 전체 적용하거나, 각 라우터에서 필요한 경로에만 적용 (기존 패턴 유지).
3. 기존 routes.ts의 `createRouter(asgardRoot)` 시그니처를 `routes/index.ts`에서 동일하게 유지 — index.ts가 변경 최소화.
4. `ALLOWED_SKILLS`, `ValidationCheck` 등 helper는 `document.routes.ts`에 배치 (skill execute에서만 사용).
5. 각 라우터 파일의 import는 `../shared/types`가 아닌 `../../shared/types`가 됨 (디렉토리 깊이 증가).
