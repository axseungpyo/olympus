# TP-008: Skill 실행 UI — 대시보드에서 TP 생성 및 위임 트리거

## Metadata
- Agent Target: codex
- Complexity: complex
- Priority: P2
- Created: 2026-03-09

## Objective

Yggdrasil SkillsPanel에 인터랙티브 실행 기능을 추가하여, 대시보드에서 직접 Skill을 트리거할 수 있게 한다.

## Scope In

### 1. 서버 API 추가
- `POST /api/skill/execute` 엔드포인트 추가 (routes.ts)
  - body: `{ skill: string, args: string }`
  - 지원 skill: `status`, `validate`
  - 실행 결과를 JSON으로 반환
  - 보안: skill명을 화이트리스트로 검증 (임의 명령 실행 방지)

### 2. SkillsPanel 업데이트
- 각 Skill 카드에 "Run" 버튼 추가 (status, validate만 활성화)
- 클릭 시 args 입력 모달/인라인 폼 표시
- 실행 결과를 카드 하단에 표시
- 실행 중 로딩 상태 표시

### 3. QuickActions 컴포넌트
- `dashboard/components/QuickActions.tsx` 생성
- Overview 탭 상단에 배치
- 버튼: "Check Status", "Validate All TPs"
- 각 버튼은 `/api/skill/execute` 호출

## Scope Out
- /delegate, /plan 등 위험한 Skill은 UI에서 실행하지 않는다 (화이트리스트 제한)
- WebSocket 기반 실시간 실행 스트리밍은 하지 않는다 (단순 요청/응답)
- 새 npm 패키지 설치하지 않는다

## Relevant Files
- `src/yggdrasil/server/routes.ts`
- `src/yggdrasil/dashboard/components/SkillsPanel.tsx`
- `src/yggdrasil/dashboard/app/page.tsx`

## Acceptance Criteria
1. `POST /api/skill/execute`가 `{ skill: "status", args: "" }`에 JSON 응답을 반환한다
2. 화이트리스트에 없는 skill 요청 시 403 에러를 반환한다
3. SkillsPanel에서 status, validate Skill 카드에 Run 버튼이 표시된다
4. `QuickActions.tsx` 파일이 존재하고 Overview 탭에 렌더링된다
5. `npm test` 20개 테스트 전부 PASS한다
6. TypeScript 컴파일 에러가 없다

## Implementation Notes
- routes.ts에서 skill 실행은 `child_process.exec`가 아닌 파일 읽기 기반으로 구현
  - `status`: INDEX.md 파싱 → 현황 JSON 반환
  - `validate`: handoff/TP-*.md 파일들 읽어서 포맷 검증 결과 반환
- 화이트리스트: `const ALLOWED_SKILLS = ["status", "validate"]`
- Pino logger 사용 (server/logger.ts)
