# RP-020: Core Domain Layer + Repository Pattern

## Summary
`server/core/`에 순수 엔티티와 포트 인터페이스를 추가하고, 파일 I/O와 프로세스 실행을 `server/adapters/`로 분리했다.
기존 `domain/`, `routes/`, `websocket/`, `watcher`, `server/index.ts`는 Container 기반 의존성 주입으로 전환해 `asgardRoot` 직접 결합을 줄였다.

## Known Issues / Follow-ups
- 현재 실행 환경에서 TCP listen 권한이 막혀 있어 `/api/health` 및 전체 HTTP 엔드투엔드 검증을 완료할 수 없었다.
- `document.routes.ts`, `health.routes.ts`, `watcher.ts`, `mcp-manager.ts` 등 일부 non-adapter 레이어에는 아직 Node.js built-in 의존성이 남아 있다. TP-020 범위에서는 core/domain I/O 추상화와 container 주입을 우선 적용했다.
- `FileSkillRegistry`는 TP 범위를 맞추기 위해 adapter로 추가됐지만, validate/status 외 스킬은 기존 동작 수준을 유지하는 래퍼 형태다.

## Recommended Next Actions
1. sandbox 제한이 없는 환경에서 `node --import tsx server/index.ts` 후 `/api/health` 및 주요 API smoke test를 실행한다.
2. TP-021에서 use case 레이어를 추가하면서 `document`, `metrics`, `mcp` 경로의 남은 built-in 의존성도 adapter 쪽으로 더 밀어낸다.
3. route/websocket 수준의 통합 테스트를 추가해 Breaking Change 검증을 자동화한다.

## Context Digest (for Odin)
- Architecture: `core (entities/ports) -> adapters (fs/process/skill) -> domain (orchestration) -> routes/websocket` 흐름으로 재정렬했다.
- Key entry points: `server/di/container.ts`, `server/domain/odin/odin-channel.ts`, `server/domain/agents/agent-control.ts`, `server/routes/index.ts`, `server/index.ts`.
- Config/env: 새 환경변수는 없고, 기존 `ASGARD_ROOT`/`PORT` 흐름을 유지한다.
- Gotchas: `tsx` CLI는 sandbox에서 IPC pipe 생성으로 막혔고, 직접 `node --import tsx server/index.ts`로 우회해도 포트 listen 자체가 `EPERM`으로 실패했다.
