# TP-010: 롤백 메커니즘 — 실패 시 git revert 기반 자동 롤백

## Metadata
- Agent Target: codex
- Complexity: medium
- Priority: P2
- Created: 2026-03-09

## Objective

Brokkr(Codex)가 blocked 상태로 종료되거나 RP가 거부(rejected)될 때, 해당 TP의 변경사항을 자동으로 git revert하는 메커니즘을 구축한다.

## Scope In

### 1. 롤백 유틸리티 (`scripts/rollback.sh`)
- 사용법: `bash scripts/rollback.sh TP-NNN`
- 동작:
  1. git log에서 해당 TP-NNN을 커밋 메시지에 포함하는 커밋들을 찾는다
  2. 해당 커밋들을 역순으로 `git revert --no-commit`으로 되돌린다
  3. 하나의 revert 커밋으로 묶어 `git commit -m "revert: rollback TP-NNN changes"`
  4. 롤백 결과를 `artifacts/logs/rollback.log`에 기록
- 에러 처리:
  - TP 관련 커밋이 없으면 "No commits found for TP-NNN" 출력 후 exit 1
  - revert 충돌 시 `git revert --abort` 후 "Rollback conflict, manual intervention needed" 출력 후 exit 2
  - dry-run 옵션: `--dry-run` 플래그로 실제 revert 없이 대상 커밋만 표시

### 2. delegate-codex.sh 연동
- Brokkr가 blocked로 종료 시 자동 롤백 제안 메시지 출력
  - "Brokkr failed. Run `bash scripts/rollback.sh TP-NNN` to revert changes."
  - 자동 실행은 하지 않음 (Odin 판단 필요)

### 3. `/rollback` Skill 정의 (`.claude/skills/rollback/SKILL.md`)
- Odin이 `/rollback TP-NNN`으로 호출
- 내부적으로 `bash scripts/rollback.sh TP-NNN` 실행
- 결과를 사용자에게 보고

### 4. 대시보드 연동
- QuickActions에 "Rollback Last Failed" 버튼 추가는 하지 않음 (위험한 작업이므로 CLI만)
- 단, `/api/skill/execute`의 화이트리스트에 `rollback`은 추가하지 않음 (보안)

## Scope Out
- 자동 롤백 (사람 판단 없이 실행)은 하지 않는다
- 원격 push된 커밋의 force push는 하지 않는다
- INDEX.md 상태 자동 변경은 하지 않는다 (수동)
- 새 npm 패키지를 설치하지 않는다

## Relevant Files
- `scripts/delegate-codex.sh` — blocked 종료 시 롤백 안내 메시지 추가
- `scripts/rollback.sh` — 신규 생성
- `.claude/skills/rollback/SKILL.md` — 신규 생성

## Acceptance Criteria
1. `scripts/rollback.sh`가 존재하고 실행 가능하다 (chmod +x)
2. `bash scripts/rollback.sh --dry-run TP-999`가 "No commits found" 메시지를 출력한다
3. `bash scripts/rollback.sh --dry-run TP-009`가 관련 커밋 목록을 표시한다
4. `.claude/skills/rollback/SKILL.md`가 존재한다
5. `delegate-codex.sh`에서 blocked 종료 시 롤백 안내 메시지가 포함된다
6. TypeScript 컴파일 에러가 없다

## Implementation Notes
- git log 검색: `git log --oneline --all --grep="TP-NNN"` 사용
- revert 순서: 최신 커밋부터 역순으로 revert (LIFO)
- rollback.log 형식: `YYYY-MM-DD HH:MM [rollback] TP-NNN: reverted commits <hash1>, <hash2>`
- SKILL.md는 기존 retry/SKILL.md 구조 참조
