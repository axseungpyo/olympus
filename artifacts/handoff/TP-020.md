## Objective

서버의 순수 도메인 레이어(core/)를 생성하고, 기존 domain/ 코드의 파일 I/O를 Repository 인터페이스 뒤로 추상화하여 Clean Architecture의 Dependency Rule을 적용한다.

## Scope In

### 1. core/entities/ — 순수 도메인 모델 (외부 의존성 제로)
- `Agent.ts` — AgentEntity 타입, AgentStatus, AgentMode, AGENT_MODES 상수
- `Task.ts` — TaskEntity 타입, TaskStatus, CreateTaskInput, UpdateTaskInput
- `Skill.ts` — SkillDefinition, SkillMatch, SKILL_PATTERNS 상수
- `Message.ts` — OdinMessage, OdinAction, CommandResult 타입

> 기존 `shared/types.ts`의 서버 전용 타입을 entities/로 이동.
> `shared/types.ts`는 클라이언트-서버 공유 DTO만 유지 (re-export 가능).

### 2. core/ports/ — 인터페이스 정의 (Dependency Inversion)
- `ITaskRepository.ts`
  ```typescript
  interface ITaskRepository {
    list(): Promise<{ active: TaskEntity[]; completed: CompletedTask[] }>
    getById(id: string): Promise<TaskDetail | null>
    create(input: CreateTaskInput): Promise<{ id: string; task: TaskEntity }>
    updateStatus(id: string, status: TaskStatus): Promise<TaskEntity | null>
    delete(id: string): Promise<{ success: boolean; message: string }>
    getNextTPNumber(): Promise<number>
  }
  ```
- `IAgentRepository.ts`
  ```typescript
  interface IAgentRepository {
    getHealth(agent: AgentName): Promise<AgentHealth>
    getStates(tasks: TaskEntity[]): Promise<AgentState[]>
    readPid(agent: AgentName): Promise<number | null>
    writePid(agent: AgentName, pid: number): Promise<void>
    deletePid(agent: AgentName): Promise<void>
  }
  ```
- `IMessageRepository.ts`
  ```typescript
  interface IMessageRepository {
    getMessages(limit?: number): OdinMessage[]
    addMessage(msg: Omit<OdinMessage, "id" | "timestamp">): OdinMessage
    loadHistory(): Promise<void>
    saveHistory(): Promise<void>
  }
  ```
- `IProcessGateway.ts`
  ```typescript
  interface IProcessGateway {
    spawn(script: string, args: string[], env: Record<string, string>): SpawnResult
    isAlive(pid: number): boolean
    kill(pid: number, signal?: string): boolean
  }
  ```
- `ISkillRegistry.ts`
  ```typescript
  interface ISkillRegistry {
    match(message: string): SkillMatch | null
    execute(skill: string, args: string): Promise<string>
    listSkills(): SkillDefinition[]
  }
  ```

### 3. adapters/repositories/ — FileSystem 구현체
- `FileTaskRepository.ts` — implements ITaskRepository
  - 기존 `domain/tasks/task-manager.ts`의 파일 I/O 로직 추출
  - `task-parser.ts` 로직 내장 (INDEX.md 파싱)
- `FileAgentRepository.ts` — implements IAgentRepository
  - 기존 `domain/agents/agent-state.ts`의 PID 파일 I/O 추출
- `FileMessageRepository.ts` — implements IMessageRepository
  - 기존 `domain/odin/odin-channel.ts`의 JSONL 영속화 추출

### 4. adapters/gateways/
- `ChildProcessGateway.ts` — implements IProcessGateway
  - 기존 `domain/agents/agent-control.ts`의 spawn/kill 로직 추출

### 6. 기존 코드 마이그레이션
- `domain/tasks/task-manager.ts` → FileTaskRepository를 사용하도록 리팩토링 (또는 adapter wrapper)
- `domain/agents/agent-control.ts` → ChildProcessGateway를 사용하도록 리팩토링
- `domain/agents/agent-state.ts` → FileAgentRepository를 사용하도록 리팩토링
- `domain/odin/odin-channel.ts` → FileMessageRepository + ISkillRegistry를 사용하도록 리팩토링
- `routes/*.routes.ts` → Container에서 의존성을 받아서 사용 (asgardRoot 대신 repository)

### 7. server/index.ts 수정
- Container 생성 후 routes에 주입
- 기존 asgardRoot 직접 전달 → Container 전달로 전환

## Acceptance Criteria
- [ ] `core/entities/` 내 파일에 fs, path, child_process import 없음 (순수 TypeScript만)
- [ ] `core/ports/` 내 파일이 모두 interface/type만 포함 (구현 코드 없음)
- [ ] `adapters/repositories/`의 각 파일이 대응하는 port 인터페이스를 implements
- [ ] `di/container.ts`가 모든 의존성을 조립하고 Container 타입을 export
- [ ] `routes/*.routes.ts`가 Container(또는 repository)를 인자로 받음 (asgardRoot 직접 의존 최소화)
- [ ] 기존 domain/ 파일이 새 repository를 통해 동작 (기능 변경 없음)
- [ ] `tsc --noEmit` 에러 0개
- [ ] `vitest run` 전체 PASS
- [ ] `curl localhost:7777/api/health` 정상 응답
- [ ] 모든 API 엔드포인트 기존과 동일하게 동작 (Breaking Change 없음)

## Implementation Notes

### Dependency Flow (After)
```
routes/*.routes.ts (HTTP)
  └→ Container.taskRepository / agentRepository / ...
      └→ FileTaskRepository (adapters/)
          └→ fs.readFile, fs.writeFile (Node.js built-in)

core/entities/ ← 의존성 없음 (가장 안쪽)
core/ports/    ← entities만 의존
adapters/      ← ports + 외부 라이브러리 의존
routes/        ← adapters 또는 ports 인터페이스 의존
```

### 마이그레이션 전략
1. core/ 디렉토리에 entities와 ports 먼저 생성 (순수 타입)
2. adapters/ 디렉토리에 구현체 생성 (기존 코드에서 추출)
3. di/container.ts 생성
4. routes를 Container 기반으로 전환
5. domain/ 파일을 adapters/ 또는 core/를 사용하도록 수정
6. 기존 domain/ 파일은 유지하되 내부적으로 새 구조를 사용

### 주의사항
- `shared/types.ts`는 클라이언트가 사용하므로 Breaking Change 금지. core/entities/에서 정의 후 shared/types.ts에서 re-export하는 방식 사용.
- routes 함수 시그니처가 변경되므로 server/index.ts도 함께 수정.
- 테스트 파일이 있다면 import 경로 업데이트 필요.
