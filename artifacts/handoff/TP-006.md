# TP-006: Yggdrasil 구조화된 로깅 — Pino 도입

## Metadata
- Agent Target: codex
- Complexity: moderate
- Priority: P1
- Created: 2026-03-09

## Objective

Yggdrasil 서버의 모든 `console.log/warn/error` 호출을 Pino 로거로 교체하여 JSON 구조화 로깅을 도입한다.

## Scope In
- `pino` 패키지 설치 (devDependency가 아닌 dependency)
- `server/logger.ts` 생성: Pino 인스턴스 + child logger 팩토리
- `server/index.ts`: 모든 console.log → logger 사용
- `server/watcher.ts`: log() 헬퍼 → Pino child logger로 교체
- `server/routes.ts`: console.error/warn → logger 사용
- `server/agents.ts`: console.error → logger 사용
- `server/parser.ts`: console.error → logger 사용
- 개발 모드에서는 `pino-pretty`로 사람이 읽기 좋은 포맷 출력
- 프로덕션 모드에서는 JSON 포맷 출력

## Scope Out
- 프론트엔드(dashboard/) 코드는 변경하지 않는다
- 로그 파일 출력 설정은 하지 않는다 (stdout만)
- 기존 테스트 파일은 수정하지 않는다 (테스트가 깨지지 않도록 주의)

## Relevant Files
- `src/yggdrasil/server/index.ts`
- `src/yggdrasil/server/watcher.ts`
- `src/yggdrasil/server/routes.ts`
- `src/yggdrasil/server/agents.ts`
- `src/yggdrasil/server/parser.ts`
- `src/yggdrasil/package.json`

## Acceptance Criteria
1. `pino`와 `pino-pretty`가 package.json에 추가되어 있다
2. `server/logger.ts` 파일이 존재하고, Pino 인스턴스를 export한다
3. `server/` 디렉토리에 `console.log`, `console.warn`, `console.error` 직접 호출이 0건이다
4. `NODE_ENV=development`에서 pino-pretty 포맷으로 출력된다
5. `npm test` (vitest) 20개 테스트 전부 PASS한다
6. 서버 기동 시 JSON 또는 pretty 로그가 정상 출력된다

## Implementation Notes
- `watcher.ts`의 기존 `log()` 헬퍼 함수를 Pino child logger로 대체
- logger.ts 예시: `export const logger = pino({ level: 'info', transport: isDev ? { target: 'pino-pretty' } : undefined })`
- child logger 패턴: `const log = logger.child({ component: 'Watcher' })`
