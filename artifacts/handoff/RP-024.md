# RP-024: Tool Use - 파일/CLI 직접 조작 + Tool Executor 패턴

## Summary
`IFileSystem`/`IToolExecutor` 포트를 추가하고, Node.js 파일시스템 어댑터와 두 개의 tool executor를 도입했다.
`ProcessCommandUseCase`는 하드코딩 switch 대신 executor 체인으로 일반 tool을 처리하고, 승인 기반 tool만 직접 유지한다.
새 파일/검색/Saga 검토 tool과 경로 안전장치, 전용 테스트를 추가했고 기존 테스트는 모두 유지했다.

## Known Issues / Follow-ups
- 서버 기동 검증은 코드 문제라기보다 현재 실행 환경 제약의 영향을 받았다. 포트 바인드 제한과 파일 watcher `EMFILE` 때문에 `/api/health`는 실제 런타임 환경에서 재검증이 필요하다.
- 사용자 지시의 `PANTHEON.md`는 현재 워크스페이스에서 찾지 못했다. RP 형식은 저장소의 `AGENTS.md`에 있는 Saga 형식을 따랐다.
- `review_saga`는 TP 요구를 만족시키기 위해 파일시스템 executor에 포함했다. TP의 두 executor 구조는 유지했고 승인 기반 tool은 `ProcessCommandUseCase`에 남겼다.
