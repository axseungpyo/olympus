## Known Issues
- `npm run build` 시 Next.js가 상위 디렉터리의 별도 lockfile을 감지해 workspace root 추론 경고를 출력한다. 빌드는 성공했으며 이번 TP 범위 밖이라 수정하지 않았다.

## Recommended Next Actions
1. 실제 대시보드에서 `flow`/`stats` 탭의 반응형 레이아웃과 샘플 chronicle 데이터 표시를 시각적으로 점검한다.
2. 필요하면 후속 TP에서 에이전트 문자열 정규화(`codex`/`brokkr`, `gemini`/`heimdall`)를 공용 유틸로 추출해 컴포넌트 간 중복 매핑을 제거한다.

## Context Digest
- Architecture: 대시보드 클라이언트 UI에 탭 기반 뷰 2종(`flow`, `stats`)이 추가되었고 서버 코드는 변경하지 않았다.
- Key entry points: `src/yggdrasil/dashboard/app/page.tsx`, `src/yggdrasil/dashboard/components/FlowView.tsx`, `src/yggdrasil/dashboard/components/StatsPanel.tsx`
- Config/env: 새 패키지 설치나 설정 변경은 없고 기존 `TASK_STATUS_CONFIG`, `AGENT_CONFIG`만 재사용했다.
- Gotchas: task의 `agent` 값이 `codex`/`gemini` 또는 정식 agent name일 수 있어 UI 컴포넌트 내부에서 양쪽 모두 매핑하도록 처리했다.
