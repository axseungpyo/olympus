# RP-009: TP 의존성 그래프 - depends-on 필드 + DAG 실행 순서

## Summary
TP Metadata의 `Depends On` 필드를 파싱해 DAG를 구성하는 서버 로직을 추가했고, 실행 순서 계산과 순환 참조 감지를 구현했다.
`GET /api/dependency-graph` 엔드포인트와 validate 순환 검사 연동을 넣었고, 대시보드 Flow 탭에 `DependencyView`를 추가해 상태 색상과 실행 가능 TP를 시각화했다.

## Known Issues
- 현재 저장소의 TP 파일들은 `Depends On`을 선언하지 않으므로 실제 그래프는 단일 병렬 실행 그룹으로 표시된다.
- 샌드박스 제한 때문에 로컬 소켓/loopback 기반 실서버 호출은 불가했고, 동일 라우트 핸들러 직접 호출로 API 응답을 검증했다.
- `npm run build`는 성공하지만 Next.js가 상위 lockfile을 보고 workspace root warning을 출력한다.

## Recommended Next Actions
1. 이후 생성되는 TP Metadata에 `Depends On`을 실제로 선언해 DAG 시각화와 실행 가능 하이라이트를 활용한다.
