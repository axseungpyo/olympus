## Objective
server/ 플랫 파일들을 domain/ (비즈니스 로직)과 infra/ (인프라 어댑터) 구조로 재배치한다. 기능 변경 없이 구조만 변경.

## Scope In

### domain/ 생성 (비즈니스 로직 — 프레임워크 import 금지)
- `server/domain/agents/agent-state.ts` ← `server/agents.ts` 이동
- `server/domain/agents/agent-control.ts` ← `server/control.ts` 이동
- `server/domain/tasks/task-manager.ts` ← `server/task-manager.ts` 이동
- `server/domain/tasks/task-parser.ts` ← `server/parser.ts` 이동
- `server/domain/odin/odin-channel.ts` ← `server/odin-channel.ts` 이동 (분할하지 않음, 이름 유지)
- `server/domain/mcp/mcp-manager.ts` ← `server/mcp-manager.ts` 이동

### infra/ 생성 (인프라 어댑터)
- `server/infra/auth.ts` ← `server/auth.ts` 이동
- `server/infra/logger.ts` ← `server/logger.ts` 이동
- `server/infra/watcher.ts` ← `server/watcher.ts` 이동
- `server/infra/metrics.ts` ← `server/metrics.ts` 이동
- `server/infra/dependency.ts` ← `server/dependency.ts` 이동

### import 경로 업데이트
- `server/routes/*.ts` — 모든 import를 새 경로로 변경
- `server/index.ts` — import 경로 변경
- `server/domain/` 내부 파일 간 상대 경로 업데이트
- `server/infra/` 내부 파일 간 상대 경로 업데이트
- `server/__tests__/*.ts` — 테스트 파일 import 경로 업데이트

### 기존 파일 삭제
- `server/agents.ts`, `server/control.ts`, `server/task-manager.ts`, `server/parser.ts`, `server/odin-channel.ts`, `server/mcp-manager.ts`, `server/auth.ts`, `server/logger.ts`, `server/watcher.ts`, `server/metrics.ts`, `server/dependency.ts` 삭제

## Implementation Notes
1. domain/ 파일은 `../../shared/types` 대신 `../../../shared/types` (깊이 +1). 또는 infra에서 re-export 패턴 사용.
2. infra/logger.ts는 domain/ 파일에서 많이 사용되므로 `../infra/logger` 또는 `../../infra/logger` 경로 사용.
3. domain/ 파일 간 의존: agents/agent-state.ts가 agents/agent-control.ts의 getRunningAgents를 import — 같은 도메인 내이므로 허용.
4. 테스트 파일의 import 경로도 반드시 변경 (예: `"../agents"` → `"../domain/agents/agent-state"`).
5. routes/ 파일의 import 깊이가 변경됨: `"../agents"` → `"../domain/agents/agent-state"`.
