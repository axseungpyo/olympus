# TP-009: TP 의존성 그래프 — depends-on 필드 + DAG 실행 순서

## Metadata
- Agent Target: codex
- Complexity: complex
- Priority: P2
- Created: 2026-03-09

## Objective

Rune(TP) 간 의존성을 선언하고, 의존성 그래프(DAG)를 기반으로 실행 순서를 판별하는 시스템을 구축한다.

## Scope In

### 1. TP 포맷 확장
- TP-NNN.md의 Metadata 섹션에 `- Depends On: TP-005, TP-006` 형식의 선택적 필드 추가
- 빈 값 또는 필드 생략 시 의존성 없음으로 처리

### 2. 의존성 파서 (`server/dependency.ts`)
- `parseDependencies(tpContent: string): string[]` — TP 본문에서 depends-on 추출
- `buildDependencyGraph(tpFiles: TPMeta[]): DependencyGraph` — 전체 TP에서 DAG 구축
- `getExecutionOrder(graph: DependencyGraph): string[][]` — 위상 정렬(Topological Sort)로 실행 가능 순서 반환 (단계별 병렬 실행 가능 그룹)
- `detectCycle(graph: DependencyGraph): string[] | null` — 순환 참조 감지
- 타입: `interface DependencyGraph { nodes: Map<string, string[]> }` 또는 적절한 형태

### 3. API 엔드포인트
- `GET /api/dependency-graph` — 전체 TP 의존성 그래프 반환
  - 응답: `{ nodes: { id: string, dependsOn: string[], status: string }[], executionOrder: string[][], hasCycle: boolean }`
- 기존 `/api/skill/execute`의 `validate` skill에 순환 참조 감지 추가

### 4. 대시보드 시각화 (`dashboard/components/DependencyView.tsx`)
- Flow 탭 하위 또는 별도 탭에 DAG 시각화
- 각 TP를 노드로, depends-on을 화살표로 표시
- CSS/SVG 기반 (외부 라이브러리 금지)
- 노드 색상: 상태별 (done=green, in-progress=amber, draft=gray, blocked=red)
- 실행 가능 TP(의존성 모두 충족)를 하이라이트

### 5. 단위 테스트
- `server/__tests__/dependency.test.ts` 생성
- 테스트 케이스:
  - 의존성 없는 TP 파싱
  - 다중 의존성 파싱
  - DAG 위상 정렬 (선형)
  - DAG 위상 정렬 (병렬 그룹)
  - 순환 참조 감지
  - 빈 그래프 처리

## Scope Out
- TP 생성 시 자동 의존성 제안 기능은 하지 않는다
- 의존성 기반 자동 실행(delegation)은 하지 않는다 — 순서 표시만
- 기존 TP-001~TP-008에 depends-on을 소급 적용하지 않는다
- 새 npm 패키지를 설치하지 않는다

## Implementation Notes
- `parseDependencies`는 정규식으로 `- Depends On:` 줄을 찾고, 쉼표로 split
- 위상 정렬: Kahn's algorithm (BFS) 사용 권장 — 순환 감지 자연스러움
- DependencyView 레이아웃: 단계(level)별 좌→우 또는 상→하 배치, 같은 단계 노드는 수직/수평 정렬
- Pino logger 사용 (server/logger.ts의 createLogger)
